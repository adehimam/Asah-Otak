6.1

Integrasikan Claude API ke Proyekmu

Setelah Fase 5, kamu udah bisa kirim request ke Claude API. Sekarang saatnya integrasiin itu ke proyek yang udah berjalan.

Langkah umum integrasi API ke website HTML statis:

1. Buat backend sederhana (bisa pakai Netlify Functions yang gratis) — ini yang "nyimpen" API Key dengan aman dan yang ngobrol langsung sama Claude API.
2. Frontend (HTML/JS) kamu ngirim request ke backend kamu sendiri, bukan langsung ke Anthropic.
3. Backend kamu yang forward ke Claude API dan balik hasilnya ke frontend.

ℹ️ Kenapa perlu backend? Supaya API Key kamu aman dan gak kelihatan di source code frontend yang bisa dilihat semua orang.
⚡ Contoh Netlify Function Sederhana
// File: netlify/functions/claude.js
exports.handler = async (event) => {
  const { prompt } = JSON.parse(event.body);

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": process.env.ANTHROPIC_API_KEY, // disimpan di env Netlify
      "anthropic-version": "2023-06-01"
    },
    body: JSON.stringify({
      model: "claude-haiku-4-5-20251001",
      max_tokens: 500,
      messages: [{ role: "user", content: prompt }]
    })
  });

  const data = await response.json();
  return {
    statusCode: 200,
    body: JSON.stringify({ result: data.content[0].text })
  };
};

6.3

Iterasi — Output Bagus atau Jelek? Perbaiki!

Setelah fitur AI kamu jalan, tugasmu belum selesai. Sekarang kamu harus evaluasi: apakah output Claude udah sesuai ekspektasi? Kalau belum — di mana masalahnya?

Biasanya masalahnya ada di prompt yang dikirim ke Claude. Ini yang disebut prompt engineering for production — ngoptimasi prompt supaya hasilnya konsisten bagus untuk banyak user yang beda-beda.

⚠️ Common issues yang sering muncul:
- Output terlalu panjang / terlalu pendek → tambah instruksi panjang di system prompt
- Format gak konsisten → kasih contoh format yang diharapkan
- Kadang bagus kadang jelek → prompt-mu terlalu ambigu, perlu lebih spesifik
- Bahasa gak sesuai → specify bahasa dan tone di system prompt

Tolong bantu improve prompt-nya supaya outputnya lebih konsisten dan sesuai ekspektasi."

6.5

Handling Error dan Edge Cases

Di dunia nyata, gak semua berjalan mulus. Claude bisa timeout, API bisa error, atau user bisa input hal-hal yang gak terduga. Kamu harus siap handle ini.

Error yang paling sering muncul di Claude API:

429 Too Many Requests — kamu kirim terlalu banyak request dalam waktu singkat. Solusi: tambah delay atau implement rate limiting.
401 Unauthorized — API Key salah atau expired. Cek key kamu.
400 Bad Request — ada yang salah di struktur request kamu. Cek format JSON-nya.
500 Internal Server Error — masalah di sisi Anthropic. Tunggu dan coba lagi.

ℹ️ Best practice: selalu wrap API call kamu dalam try-catch, dan kasih feedback yang jelas ke user kalau terjadi error — jangan biarkan user stuck tanpa penjelasan.
