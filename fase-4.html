💻 FASE 4

Claude untuk Coding

Gak harus jago coding dulu buat bisa pakai Claude buat ngoding. Di fase ini kita belajar cara kerja bareng Claude sebagai pair programmer.

4.1

Minta Claude Jelaskan Kode yang Kamu Gak Ngerti

Ini entry point terbaik buat yang masih awam coding. Kamu gak harus ngerti semua kode yang kamu pakai — yang penting kamu tau apa fungsinya dan gimana cara kerjanya.

Tinggal paste kodenya ke Claude, terus tanya: "Ini ngapain? Jelaskan baris per baris pakai bahasa awam."

💡 Tips: Kasih konteks bahasa pemrogramannya. Misalnya: "Ini kode JavaScript, tolong jelaskan..." — supaya Claude gak salah interpretasi.
⚡ Coba Langsung
Ini kode JavaScript yang aku temukan. Tolong jelaskan cara kerjanya baris per baris pakai bahasa awam yang gampang dimengerti:

const btn = document.querySelector("#submitBtn");
btn.addEventListener("click", async () => {
  const val = document.querySelector("#inputText").value;
  if (!val.trim()) return alert("Isi dulu inputnya!");
  const res = await fetch("/api/process", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ text: val })
  });
  const data = await res.json();
  document.querySelector("#output").textContent = data.result;
});

4.2

Debug Error — Cara Paste Error yang Bener ke Claude

Ini skill paling sering dipakai developer, termasuk yang udah senior. Kamu nemuin error? Tinggal tanya Claude. Tapi ada cara paste yang bener supaya Claude bisa bantu maksimal.

Yang perlu kamu kasih ke Claude: (1) pesan error lengkapnya, (2) kode yang menyebabkan error, (3) konteks: kamu lagi ngapain, pakai framework apa, dan apa yang udah kamu coba.

⚠️ Jangan cuma paste pesan error-nya doang tanpa kode dan konteks. Claude bakal susah bantu karena informasinya kurang.
⚡ Template Debug yang Efektif
Aku dapat error ini waktu jalanin kode:
[PASTE PESAN ERROR DI SINI]

Ini kodenya:
[PASTE KODE DI SINI]

Konteks:
- Aku lagi bikin: [jelaskan fungsinya]
- Pakai: [bahasa/framework/library apa]
- Yang udah aku coba: [sebutkan kalau ada]

Tolong bantu debug dan jelaskan kenapa errornya bisa terjadi.
Buka Claude → Coba Sekarang
4.3

Minta Claude Bikin Fitur Baru dari Kode yang Ada

Kamu punya kode yang udah jalan, dan mau tambah fitur baru? Kasih Claude kode kamu yang sekarang, jelaskan fitur yang mau ditambahin, dan minta dia tulis kode tambahannya.

Kuncinya: kasih full context. Jangan cuma bilang "tambah fitur X" tanpa kasih kode yang udah ada — Claude bakal nulis kode yang mungkin gak kompatibel sama struktur project kamu.

ℹ️ Kalau project kamu besar, cukup kasih file yang relevan aja — bukan semua file. Misalnya kalau mau tambah fitur di halaman login, kasih aja kode halaman login-nya.
⚡ Template Tambah Fitur
Ini kode HTML/JS yang udah aku punya:
[PASTE KODE YANG ADA]

Aku mau tambah fitur: [JELASKAN FITUR BARU]

Syarat tambahannya:
- [syarat 1, misalnya: jangan ubah struktur yang udah ada]
- [syarat 2, misalnya: pakai vanilla JS, jangan tambah library baru]

Tolong tulis kode tambahannya dan tunjukkan di mana harus ditaruh.
Buka Claude → Coba Sekarang
4.4

Code Review Bareng Claude

Kode kamu udah jalan, tapi mau mastiin udah bagus? Minta Claude review. Dia bisa kasih feedback soal: potensi bug, performa, keamanan, dan keterbacaan kode (readability).

Ini berguna banget buat self-taught developer kayak kamu, sob — karena kamu gak punya senior dev yang bisa review kode kamu langsung.

💡 Minta review yang spesifik: Daripada "review kode ini", lebih baik "review kode ini dengan fokus ke: potensi XSS vulnerability dan apakah ada cara yang lebih efisien."
⚡ Template Code Review
Tolong review kode ini dan kasih feedback:

[PASTE KODE DI SINI]

Yang aku minta diperhatikan:
1. Ada potensi bug atau error yang mungkin belum aku sadar?
2. Ada bagian yang bisa ditulis lebih efisien?
3. Kalau ini nanti diakses banyak user, ada potensi masalah performa?

Kasih penjelasan untuk setiap poin yang kamu temukan.
Buka Claude → Coba Sekarang
4.5

Claude sebagai Pair Programmer Harian

Pair programming itu konsep di mana dua programmer kerja bareng di kode yang sama — satu yang nulis, satu yang review dan kasih saran. Claude bisa jadi "pasangan coding" kamu yang selalu available.

Caranya: setiap kali mulai sesi coding, kasih Claude konteks project kamu dulu. Terus selama coding, tanya apapun — dari hal teknis sampai "ini approach-ku udah bener gak ya?"

💡 Workflow yang bagus: Buka dua jendela — satu VS Code / editor kamu, satu Claude. Tiap nemu hambatan, langsung tanya Claude. Jangan tunggu sampai stuck lama.
⚡ Template Mulai Sesi Coding
Aku mau minta kamu jadi pair programmer aku hari ini.

Konteks project:
- Nama: [nama project]
- Stack: [teknologi yang dipakai]
- Yang lagi dikerjain hari ini: [fitur/task apa]
- File yang relevan: [sebutkan atau paste kodenya]

Aku akan tanya-tanya sambil ngoding. Kalau aku tanya sesuatu yang kamu rasa approach-nya kurang tepat, langsung kasih tau ya jangan cuma jawab pertanyaannya aja.
Buka Claude → Coba Sekarang
4.6

Latihan Nyata — Improve Salah Satu Fitur Proyekmu

Teori tanpa praktek gak akan nempel. Sekarang saatnya coba di project nyata kamu. Pilih satu fitur kecil yang mau diperbaiki atau ditambahkan ke salah satu project kamu — bisa CTS, Pix Shot, atau project lainnya.

Prosesnya: ambil kode yang relevan → paste ke Claude → jelaskan apa yang mau diperbaiki → iterasi sampai hasilnya oke → implementasikan.

ℹ️ Gak perlu langsung ambil fitur yang besar dan kompleks. Mulai dari yang kecil dulu — misalnya perbaiki tampilan tombol, atau tambah validasi input sederhana.
⚡ Coba Sekarang
Ambil kode dari project kamu sendiri (CTS, Pix Shot, atau apapun), paste ke Claude, dan minta satu improvement kecil.

Contoh prompt-nya:
"Ini kode tombol download di project aku: [paste kode]
Saat ini tombolnya langsung download tanpa konfirmasi.
Bisa tambahkan dialog konfirmasi sederhana sebelum download, tanpa library tambahan?"
